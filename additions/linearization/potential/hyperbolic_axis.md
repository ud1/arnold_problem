Программа hyperbolic-axis

1. Цель

  1. Реализовать режим распрямления, где вертикальной осью делается выбранная прямая (или перебираются все).
  2. Убрать проблему “переворота” наклонов через вращение O-матрицы.
  3. Решать задачу как LP-выполнимость с фиксированными b и переменными m через HiGHS.

2. Формат программы

  1. Новый бинарник: hyperbolic-axis (отдельный hyperbolic_axis.cpp).
  2. Базовый вход: генераторы (--gens, --gens-file, stdin filter-mode как в main.cpp/parabolic_axis.cpp).

3. Основной флаг:

  - --hyperbolic-axis K (конкретная ось),
  - без параметра: перебор всех осей подряд через вращения

3. CLI (обязательный)

  1. --gens "..." / --gens-file path.
  2. [--hyperbolic-axis K].
  3. --axis-eps E (для пары ±eps, E>0), по умолчанию 0,00001
  4. --margin M (по умолчанию 1.0).
  5. --try-reflect (опционально: дополнительно пробовать отражённую O-матрицу).
  6. --output-format compact|full|json.
  7. --metadata-only для filter-mode.

4. Ограничения входа

  1. Поддерживаются только нечетные N (иначе ошибка с понятным текстом).
  2. K в диапазоне 0..N-1.
  3. Для режима без K: попробовать все K=0..N-1.
  4. При невозможном вращении (по правилам OMatrix rotate/canRotate) попытка помечается как SKIP_UNROTATABLE.

5. Математическая модель

  1. После выбора оси выполняется вращение O-матрицы так, чтобы ось стала граничной (как в OMatrix.php::rotate).
  2. Фиксируются b на линиях, пересекающих ось, по равномерным углам с дыркой ±eps:

  - N=2p+1,
  - углы: {-k*pi/(2p), k=1..p-1} ∪ {-eps, +eps} ∪ {+k*pi/(2p), k=1..p-1},
  - b_i = tan(theta_i) в порядке пересечений на оси.

  3. Переменные оптимизации: только m_i (для всех неосевых линий; для осевой m не используется).
  4. Неравенства берутся из текущих edge-условий, но с подстановкой фиксированных b; получаем линейную систему по m:

  - A m <= rhs - margin.

  5. Решение через HiGHS:

  - phase-I: min t, A m - t <= rhs, t>=0.
  - feasible, если t <= tol и max_violation <= tol.

6. Перебор в режиме без --hyperbolic-axis без номера

  1. Для каждой оси K:

  - rotation K,
  - (опционально) reflected+rotation K,
  - построение LP, запуск HiGHS.

  2. Критерий выбора лучшего:

  - сначала feasible,
  - затем минимальный t,
  - затем минимальный max_violation.

  3. В выводе обязательно указать: исходная ось, применённая rotation, было ли reflect.

7. Вывод

  1. SAT / NO_SOLUTION.
  2. Диагностика:

  - n, gens_count, axis_input, axis_used, rotation, reflected,
  - margin, eps, t, max_violation, worst_raw.

  3. Полный режим:

  - список линий [(m_i,b_i)] в исходной нумерации (обратно развернуть rotation/reflect mapping).

  4. Filter-mode:

  - одна строка на кейс в формате, совместимом с текущим пайплайном.

8. Совместимость

  1. main.cpp не менять по поведению.
  2. Новая программа независима, но использует те же парсеры форматов входа.
  3. Сборка с HiGHS из /mnt/c/Data/git/arnold-research/code/HiGHS через явные -I/-L.

9. Точность и устойчивость

  1. Числовые пороги:

  - tol = 1e-10 для t и нарушений.

  2. Проверка решения после HiGHS:

  - явный пересчёт всех raw и worst_raw.

  3. При NaN/Inf в углах/tan — завершать кейс с ошибкой валидации.

10. Тесты (минимум)

  1. Unit:

  - rotation/reflect эквивалентны OMatrix.php на наборе кейсов.
  - генерация углов для нечетных N.

  2. Интеграция:

  - --hyperbolic-axis K на известном SAT-кейсе.
  - перебор без K выбирает SAT среди нескольких осей.
  - отказ на четном N.

  3. Regression:

  - совпадение статуса SAT/NO с ожидаемыми эталонами на наборе входов.

11. Критерии приёмки

  1. Программа компилируется и линкуется с HiGHS.
  2. Оба режима с --hyperbolic-axis и без работают.
  3. Для нечетных N корректно формируются фиксированные b.
  4. Решение идёт по LP с переменными только m.
  5. В отчёте есть информация о rotation/reflect и выбранной оси.
